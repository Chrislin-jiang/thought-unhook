# 🧠 念头解钩器（Thought Unhook）— 专业产品方案 v3.0

> **产品愿景**：你不是你的念头。我们帮助用户将脑内自动产生的声音"物化"出来——看见它、触碰它、理解它、放下它。
>
> 念头解钩器不是心理治疗工具，而是一款**日常心理卫生应用**——像刷牙一样自然、像日记一样私密、像冥想一样有效。

---

## 一、角色定位与产品概述

### 1.1 产品定义

**念头解钩器（Thought Unhook）** 是一款面向个人的**私密、治愈型心理自助应用**。它的核心是帮助用户进行：
- **情绪觉察与记录** — 结构化地捕捉和理解自己的情绪
- **念头物化与解钩** — 将脑内声音外化为可操作的视觉对象
- **AI陪伴对话** — 温暖、非评判的智慧陪伴
- **定期自我复盘** — 基于数据的个性化成长洞察

### 1.2 核心理念

| 原则 | 说明 |
|------|------|
| **不是压制，而是"看见"** | 认知解离（Cognitive Defusion）— 与念头拉开距离 |
| **不是分析，而是"体验"** | 具身认知（Embodied Cognition）— 用身体感受消解过程 |
| **不是反驳，而是"变轻"** | 不与念头争辩，只观察它自然消散 |
| **不是治疗，而是"卫生"** | 像刷牙一样日常、像呼吸一样自然 |
| **不是说教，而是"陪伴"** | AI 是共情的倾听者，不是冷冰冰的分析器 |

### 1.3 产品名

| 候选 | 风格 | 适用场景 |
|------|------|---------|
| **念头解钩器**（主打） | 功能明确，有动作感 | 品牌主名 |
| Thought Unhook | 国际化，简洁 | 英文品牌 |
| 脑内弹幕清理器 | 轻松幽默 | 年轻用户传播 |

### 1.4 目标用户画像

| 用户群体 | 核心痛点 | 典型场景 | 高频触发念头 |
|---------|---------|---------|------------|
| 💭 焦虑反刍者 | 同一个念头反复循环，越想越焦虑 | 通勤路上、独处时 | "我是不是不够好" |
| 🌙 失眠人群 | 睡前脑内嘈杂，无法安静 | 躺在床上辗转 | "明天还有一堆事" |
| 💻 高压脑力工作者 | 思绪过载，难以聚焦当下 | 工作deadline前 | "这个需求我肯定做不完" |
| 🧘 冥想初学者 | 知道要"观察念头"但不知怎么做 | 尝试冥想时走神 | "我怎么又走神了" |
| 💔 情绪敏感者 | 容易被一个想法带走，陷入情绪漩涡 | 社交冲突后 | "他是不是对我有意见" |
| 😢 需要情绪出口者 | 没人可以倾诉，需要安全空间 | 深夜独处、委屈时 | "没人理解我" |

> 👉 共同特征：这些都是**自动弹出的"脑内声音"**——不请自来、反复出现、像事实一样有说服力。

## 二、核心概念模型

### 2.1 念头的"物化"模型 — 3步解钩

```
Step 1️⃣  捕捉（Capture）     →  用户输入/说出脑内声音
Step 2️⃣  物化（Externalize）  →  系统将念头变成可感知的视觉物体
Step 3️⃣  解钩（Defuse）       →  通过交互操作拉开心理距离
```

### 2.2 念头生命周期

```
[产生] → [捕捉] → [物化] → [观察] → [理解] → [解钩/消解] → [释放]
  ↑                                                           |
  └──────────────── 念头自然循环 ←──────────────────────────────┘
```

### 2.3 念头的物理形态

每一个念头输入后，会被渲染为一个**漂浮的半透明气泡/物体**，具有以下可感知属性：

| 属性 | 含义 | 视觉表现 |
|------|------|---------|
| **大小** | 这个念头占据你多少注意力 | 气泡尺寸（越频繁出现→越大） |
| **颜色** | 念头的情绪色彩 | 暖色=焦虑/愤怒，冷色=悲伤，灰色=麻木 |
| **密度** | 念头的"粘性"（越粘越难放下） | 透明度，越实心越粘 |
| **运动** | 念头的活跃程度 | 抖动频率、漂浮轨迹 |
| **文字** | 念头的内容 | 气泡内若隐若现的文字 |
| **角色标识** | 念头的"人格"来源（AI识别） | 小图标：👿焦虑怪 / 🤖完美主义 / 👶内在小孩 |

### 2.4 八种解钩操作

基于ACT（接纳承诺疗法）经典练习，设计以下交互方式：

| # | 操作 | 心理机制 | 具体体验 |
|---|------|---------|---------|
| 1 | 🫧 **看见** | 去融合 | 念头从脑内文字变为眼前漂浮物 |
| 2 | 🏷️ **贴标签** | 分类觉察 | "这是一个关于未来的担忧" |
| 3 | ✏️ **改写** | 认知解离 | "我完了" → "我注意到我在想：我完了" |
| 4 | 🎵 **变声** | 去严肃化 | 用卡通音/慢速重新播放念头 |
| 5 | 🔍 **放大/缩小** | 透视转换 | 捏合手势让念头变大变小 |
| 6 | 💨 **吹走** | 释放 | 长按或吹气让气泡飘走 |
| 7 | 🫠 **融化** | 接纳消散 | 看着念头慢慢变透明直到消失 |
| 8 | 📌 **暂存** | 稍后处理 | 把念头放进"念头罐"，不急着处理 |

> 核心原则：**不是反驳它，而是让它"变轻"**

### 🫧 模块一：念头空间（Thought Space）— 核心快速解钩场景

**目标**：提供一个沉浸式的空间，用户在这里"倾倒"脑内念头并与之物理化互动，实现即时解钩。

**适用场景**：当用户被一个念头缠住、需要快速拉开距离时。

#### 用户流程

```
1. 打开App → 进入「念头空间」
2. 看到一片安静的沉浸空间（星空/水面/虚空，可选主题）
3. 点击 ＋ 或语音按钮 → 输入一个念头
4. AI 即时分析 → 念头以「气泡」形态出现（含情绪色彩+角色标识）
5. 可以继续输入更多念头（多个气泡漂浮）
6. 选择一个气泡 → 展开「解钩操作面板」
7. 选择操作（改写/变声/吹走/融化/缩小/贴标签/暂存）
8. 念头被释放/暂存 → 空间重归平静
```

#### 输入方式

- ✍️ **文字输入**：简短句子描述念头（"我怕明天的会议搞砸"）
- 🎤 **语音输入**：直接说出来，AI转文字（更自然的"倾倒"感）
- 🤖 **AI辅助捕捉**：输入模糊感受，AI帮你精确命名
  - "说不清的难受" → "你可能在感受一种无力感？"
  - "心里堵得慌" → "听起来像是一种压抑或委屈的感觉？"

#### 空间交互设计

- 气泡会缓缓漂浮、轻微碰撞
- 长按气泡 → 展开操作面板（8种操作）
- 双指捏合 → 缩放念头大小
- 向上滑动气泡 → 吹走/释放
- 向下拖动 → 放入念头罐暂存
- 摇一摇手机 → 所有气泡抖散（"shake off"）
- 气泡上有角色小图标（👿🤖👶等），点击查看"谁在说话"

#### 解钩操作面板

```
┌─────────────────────────────────────┐
│  "我肯定搞不定这个项目"               │
│  🔮 预言家 · 灾难化思维 · 焦虑        │  ← AI自动识别
├─────────────────────────────────────┤
│                                     │
│  ✏️ 改写    🎵 变声    🏷️ 标签      │  ← 解钩操作
│                                     │
│  💨 吹走    🫠 融化    🔍 缩小       │
│                                     │
│  📌 暂存                  ⭐ 推荐     │  ← AI个性化推荐
│                                     │
└─────────────────────────────────────┘
```

### 📝 模块二：情绪日记（Emotion Journal）— 结构化情绪记录

**目标**：引导用户完成一次**结构化的情绪记录与自我觉察**，并可随时回看之前的记录。

**适用场景**：当用户经历了一件让自己情绪波动的事情，需要停下来认真梳理时。

> 💡 设计理念：借鉴心理学中的"情绪日记"（Mood Diary）方法，采用**分步问答引导式**而非自由书写，帮助用户系统性地觉察情绪、理解背后的需求、并重新定义事件。

#### 功能入口

提供两个核心功能按钮：「✏️ 开始记录」与「📖 回看过往记录」

#### 「开始记录」功能流程

采用**分步问答引导式**。用户点击后，依次呈现以下问题，填写完一个才进入下一个：

**第一部分：记录情绪（事实与感受）**

| # | 引导问题 | 类型 | 说明 |
|---|---------|------|------|
| 1 | ⏰ 时间、📍 地点、👤 人物 | 快速标签选择 | 帮助锚定具体场景 |
| 2 | 当时发生了什么？（描述事件） | 文本输入 | 只描述事实，不加评判 |
| 3 | 当时的情绪感受/想法？ | 情绪标签选择+文本 | 如：愤怒、委屈、焦虑、无力... |
| 4 | 当时生理上有什么感觉？ | 可选标签 | 例如：胸口发紧、头疼、手心出汗、胃不舒服 |
| 5 | 情绪强度打分 | 1-5 滑动选择 | 可视化的情绪温度计 |
| 6 | 事情发生时我的反应是什么？/我做了什么？ | 文本输入 | 觉察自己的行为模式 |
| 7 | 出现这个情绪是因为我有什么需求？ | 引导选择+文本 | 如：被认可、安全感、自主权、归属感... |

**第二部分：消解情绪（反思与行动）**

| # | 引导问题 | 类型 | 说明 |
|---|---------|------|------|
| 8 | 为什么我会这样想？ | 文本输入 | 探索念头的根源 |
| 9 | 这个想法是事实吗？还是有所放大？ | 单选+文本 | 认知解离的关键步骤 |
| 10 | 如果今天别人发生这件事，我会对TA说什么？ | 文本输入 | 自我关怀视角转换 |
| 11 | 重新定义事情严重程度 | 1-5 滑动选择 | 与第5题对比，观察认知变化 |
| 12 | 如果下次发生类似的事，我可以怎么做？ | 文本输入 | 导向具体行动 |

**记录完成反馈**：
```
✅ 记录完成！
   
"你刚才的情绪强度从 4 降到了 2。
 看见情绪本身，就是在照顾自己。
 你做得很好 🌱"
```

#### 「回看过往记录」功能

- 按**时间轴倒序**浏览所有历史记录
- 每条记录以**简洁摘要**（日期、情绪关键词、事件摘要）形式呈现
- 点击可查看完整详情
- 支持按情绪类型、时间段筛选
- AI 自动识别**重复出现的情绪模式**，在时间线上标注提示

#### 情绪日记的设计原则

| 原则 | 描述 |
|------|------|
| **引导而非自由** | 通过结构化问题降低表达门槛，让不擅表达的人也能完成记录 |
| **事实先于分析** | 先记录事实和感受，再进入反思，避免过早理性化 |
| **生理觉察** | 关注身体信号（胸口发紧、头疼），增强具身体验 |
| **视角转换** | "如果是别人遇到这件事"帮助用户跳出第一人称 |
| **前后对比** | 情绪强度的两次打分让用户直观看到觉察本身的疗愈效果 |
| **行动导向** | 最后一问引向具体行动，避免停留在情绪漩涡中 |

### 💬 模块三：心事陪伴（Soul Chat）— AI 持续性对话空间

**目标**：提供一个模拟"人生导师"的持续性对话空间，让用户在需要倾诉时有一个温暖、智慧、安全的陪伴者。

**适用场景**：深夜辗转难眠、遇到情绪困境、需要被倾听但身边无人可说时。

#### AI 角色定义

**核心人格**：一位富有同理心、智慧、温和的倾听者与伙伴。

| 维度 | 定义 |
|------|------|
| **首要原则** | 优先共情，避免说教 |
| **对话风格** | 通过提问帮助用户自己梳理思路，语言自然如真人对话 |
| **角色定位** | 不是心理咨询师，不做诊断；是一个比你多了一些人生智慧的好友 |
| **语言特点** | 温暖、简洁、不堆砌大道理；适时使用隐喻和小故事 |
| **边界** | 检测到高风险内容时温和引导至专业求助资源 |

**AI 对话设计原则**：
```
1. 先接纳（"我听到你说..."），再探索（"你觉得是什么让你..."）
2. 不给标准答案，引导用户自己发现
3. 每3-5轮对话自然穿插一个小反思："你现在感觉如何？"
4. 在合适时机引导用户去尝试"念头空间"或"情绪日记"
5. 对话结束时给出温暖的收尾："今天你已经做得很好了 🌙"
```

#### 界面氛围设计

每次进入随机呈现**治愈场景**，并配有相应的沉浸式背景音效（可选）：

| 场景 | 视觉描述 | 音效 |
|------|---------|------|
| 🔥 篝火营地 | 夜色中温暖的篝火，星星点点 | 柴火噼啪声+虫鸣 |
| ⭐ 星空天台 | 高处露台上仰望满天星河 | 微风+远处城市白噪音 |
| 📚 深夜书房 | 暖光台灯、书架、一杯热茶 | 翻书声+钟摆+雨声 |
| 🌲 森林溪边 | 阳光透过树叶，溪水潺潺 | 流水声+鸟鸣+风吹树叶 |
| 🌊 海边小屋 | 面朝大海的小窗，远处灯塔 | 海浪声+海鸥 |

**交互设计**：
- 对话区域采用简洁的消息流
- 用户输入支持文字和语音
- AI 回复带有自然的"正在思考..."打字效果
- 场景背景可手动切换或设为随机
- 背景音效有独立开关和音量控制

### 💌 模块四：每周回信（Weekly Letter）— 个性化复盘与成长

**目标**：基于用户本周的情绪日记、念头记录与陪聊内容，生成一份**个性化的情感复盘信**。

**适用场景**：每周末或固定时间，给用户一份温暖的自我总结。

#### 内容形式

模拟**"一封来自树洞的来信"**，风格亲切、温暖、充满鼓励。

**信件内容要点**：

| 要素 | 说明 | 示例 |
|------|------|------|
| 📊 **情绪关键词总结** | 提取本周高频情绪词 | "这周出现最多的词是「焦虑」和「疲惫」" |
| 🔄 **模式发现** | 识别重复出现的思维/行为模式 | "你注意到了吗？周一和周四你的焦虑感特别强" |
| ✨ **自我觉察肯定** | 肯定用户本周的每一次觉察行为 | "你这周记录了 5 次情绪日记，这本身就是一种力量" |
| 🌱 **温和成长建议** | 基于数据给出非说教式建议 | "下周可以试试在焦虑来临时先做3次深呼吸" |
| 💪 **能量补给** | 结尾的温暖鼓励 | "你正在用自己的方式，慢慢变得更好。加油 ☀️" |

#### 界面设计

- 设计成**"复古邮箱"** 样式
- 新信件有**醒目提醒动画**（信封飘入效果）
- 打开信件有展开信纸的动画
- 过往信件可按时间**归档查看**
- 支持长按保存为图片/分享

#### 信件生成逻辑

```
数据来源：
├── 本周情绪日记记录
├── 念头空间的念头数据（类型、频次、解钩方式）
├── 心事陪伴的对话摘要
└── 觉察日志的统计数据

生成时机：
├── 每周日晚 20:00 自动生成
├── 用户也可手动触发"给我写封信"
└── 至少有 3 条记录才生成（数据不足时提示鼓励记录）
```

### 🧪 模块五：解钩实验室（Unhook Lab）— 引导式练习

**目标**：基于 ACT/CBT/正念 的结构化练习，以游戏化方式帮助用户学会认知解离技术。

**适用场景**：用户想要系统学习解钩技巧，或者想要安静下来做一次引导练习。

#### 练习列表（8个）

| # | 练习名 | 时长 | 机制 | 难度 |
|---|--------|------|------|------|
| 1 | 🚂 **念头列车** | 3min | 念头变成列车车厢，你是站台上的观察者，看它们驶过 | ⭐ |
| 2 | 📺 **念头电视** | 2min | 念头变成电视字幕，你手持遥控器可以调台、静音、关机 | ⭐ |
| 3 | ☁️ **云上书写** | 3min | 念头写在云朵上，风吹过，文字渐渐散去 | ⭐ |
| 4 | 🎈 **气球释放** | 2min | 每个念头绑在气球上，松手让它飞走 | ⭐ |
| 5 | 🌊 **念头漂流瓶** | 3min | 念头装进瓶子，放入海浪，看它慢慢漂远 | ⭐⭐ |
| 6 | 🔬 **念头显微镜** | 4min | "显微镜"观察念头微观结构（词语→音节→符号→像素→消散） | ⭐⭐ |
| 7 | 👿 **认识脑内角色** | 5min | 输入多个念头，AI识别你的"内在角色"阵容，逐个见面 | ⭐⭐⭐ |
| 8 | 🧘 **身体扫描觉察** | 5min | 引导关注身体各部位的感受，找到情绪在身体的"落脚点" | ⭐⭐ |

#### 每个练习的标准结构

```
引导语(文字+语音) → 输入/选择念头 → 动画交互 → 呼吸锚定 → 结束反馈
```

#### 重点练习详细设计 —「认识脑内角色」

```
1. "让我们来认识一下住在你脑子里的小伙伴们"
2. 用户连续输入 3-5 个近期反复出现的念头
3. AI分析并为每个念头分配角色
4. 动画：一个个角色从气泡中"走出来"，附带自我介绍
   - 👿 "嗨，我是焦虑怪，我的工作是让你担心一切可能出错的事"
   - 🤖 "我是完美主义AI，我觉得你永远还可以做得更好"
5. 用户可以给每个角色"起昵称"
6. 结束："现在你知道了，下次它们说话时，你可以说——
    '又是你啊，焦虑怪 👿 你想说什么？我听到了。'"
```

### 🌙 睡前模式（Night Mode）— 大脑清空仪式

**目标**：专门为睡前设计的简化体验，帮助用户清空一天的"脑内噪音"。

**流程**：
```
1. 深色界面 + 极简交互 + 低亮度自动提示
2. 提示："把今天还在脑子里转的东西都倒出来"
3. 快速连续输入念头（语音/文字，不做复杂分析）
4. 所有念头变成萤火虫般的光点
5. 配合缓慢呼吸引导（吸4秒-停4秒-呼6秒）
6. 每次呼气，一个光点缓缓熄灭（念头逐个告别）
7. 屏幕渐暗 → "晚安，你已经放下了今天 🌙"
8. 可选：播放白噪音/自然音助眠（定时关闭）
```

## 四、AI 能力设计（四层架构）

> AI 是产品的核心差异化能力，贯穿所有模块。

### 4.1 AI 四层能力模型

```
Layer 1 — 识别与分类（v0.1 MVP）
    ↓      情绪识别、认知扭曲分类、强度评估
Layer 2 — 自动解钩改写（v0.1 MVP）
    ↓      ACT 句式自动化、多种改写技术
Layer 3 — 个性化推荐（v0.3）
    ↓      基于用户画像的解钩方式推荐
Layer 4 — 念头人格化（v0.2 杀手特性）
           内在角色识别、角色对话、角色成长
```

### 🔍 Layer 1：念头识别与分类

**输入**：用户的原始念头文字
**输出**：认知扭曲类型 + 情绪标签 + 强度评估

| 用户输入 | AI识别类型 | 情绪标签 |
|---------|-----------|---------|
| "我肯定会失败" | 灾难化思维（Catastrophizing） | 焦虑 / 自我否定 |
| "他一定是讨厌我" | 读心术（Mind Reading） | 不安 / 社交焦虑 |
| "我什么都做不好" | 过度概括（Overgeneralization） | 自我否定 / 抑郁 |
| "我应该更努力才对" | 应该思维（Should Statements） | 内疚 / 自责 |
| "这全是我的错" | 个人化（Personalization） | 内疚 / 自责 |

**实现路径**：
- v0.1：基于关键词匹配 + 简单NLP规则（零成本）
- v0.2：接入 LLM API 进行深度语义分析
- 分类结果以小标签形式显示在气泡上

### ✏️ Layer 2：自动解钩改写

**核心**：将 ACT 经典的"认知解离句式"自动化。

| 原始念头 | 解钩改写 | 技术 |
|---------|---------|------|
| "我完了" | "我注意到我在想：我完了" | 觉察前缀 |
| "我肯定会失败" | "我正在产生一个关于失败的想法" | 客体化 |
| "他讨厌我" | "我的大脑在告诉我一个故事：他讨厌我" | 叙事化 |
| "我不够好" | "「我不够好」这个念头又来了，像个老朋友" | 拟人化 |

### 🎯 Layer 3：个性化解钩推荐

| 用户类型 | 特征 | 推荐解钩方式 |
|---------|------|------------|
| 理性型 | 喜欢逻辑，常用"改写" | ✏️ 改写 + 🏷️ 贴标签 |
| 情绪型 | 感受强烈，偏好体验 | 🎵 变声 + 🫠 融化 |
| 行动型 | 喜欢"做点什么" | 💨 吹走 + 🔍 缩小 |
| 视觉型 | 喜欢看变化过程 | 🫠 融化 + 解钩实验室练习 |

### 👿 Layer 4：念头人格化（杀手特性）

> 让用户发现"这些声音不是我，是不同角色在说话"——认知解离最强大的形式。

**预设角色**：

| 角色 | 图标 | 典型台词 | 深层含义 |
|------|------|---------|---------|
| 👿 焦虑怪 | 不断颤抖的小恶魔 | "万一失败了怎么办？" | 过度预警系统 |
| 🤖 完美主义AI | 冰冷的机器人 | "你还不够好，再努力" | 不合理标准 |
| 👶 内在小孩 | 蜷缩的小人 | "我好害怕，没人要我" | 早期创伤回声 |
| 🎭 表演者 | 戴面具的人 | "他们会怎么看我？" | 社会评价恐惧 |
| 📢 评判官 | 拿锤子的法官 | "这全是你的错" | 自我惩罚 |
| 🔮 预言家 | 水晶球先知 | "我已经看到结局了" | 灾难化预期 |

## 五、界面与风格设计

### 5.1 设计原则

| 原则 | 描述 |
|------|------|
| **极简沉浸** | 界面尽可能少元素，让"空间感"成为主角 |
| **温暖安全** | 色调柔和，交互无压力，没有打卡压力 |
| **物理触感** | 动画有真实的物理反馈（弹性、重力、摩擦） |
| **无评判** | 不评判念头的好坏，只帮助"看见" |
| **渐进引导** | 首次使用温和引导，不堆砌功能 |
| **AI无感化** | AI分析在后台进行，不打断用户的情感流 |

### 5.2 视觉风格

- **整体风格**：以**深蓝、暖橙、米白、暗绿**为主色调的暗色系，保护双眼且营造静谧感
- **主色调**：深蓝/深紫（安静的夜空感）
- **辅助色**：暖橙色用于温暖元素（信件、陪聊），暗绿色用于成长元素（日志、洞察）
- **点缀色**：气泡使用情绪对应色彩（半透明渐变）
- **毛玻璃效果**：卡片、面板使用 Glassmorphism 风格
- **角色图标**：小型 emoji 风格，温暖不恐怖
- **字体**：中文使用思源黑体/霞鹜文楷，英文使用 Inter
- **图标**：线性图标 + 微动效
- **背景**：可选主题（星空/海底/森林/虚空白/篝火营地）

// ===== 情绪类型 =====
type EmotionType = 
  | 'anxiety' | 'anger' | 'sadness' | 'fear'
  | 'guilt' | 'shame' | 'frustration' | 'helplessness'
  | 'loneliness' | 'grief' | 'neutral' | 'mixed';

// ===== 认知扭曲类型 =====
type CognitiveDistortion =
  | 'catastrophizing' | 'overgeneralization' | 'mind-reading'
  | 'should-statements' | 'personalization' | 'black-white'
  | 'emotional-reasoning' | 'fortune-telling' | 'labeling'
  | 'discounting-positive' | 'unknown';

// ===== 内在角色类型 =====
type PersonaType =
  | 'anxiety-monster' | 'perfectionist-ai' | 'inner-child'
  | 'performer' | 'judge' | 'fortune-teller';

// ===== 释放方式 =====
type ReleaseMethod = 
  | 'blow' | 'melt' | 'float' | 'shrink'
  | 'rewrite' | 'voice' | 'store';

// ===== 用户画像 =====
interface UserProfile {
  id: string;
  preferredMethods: ReleaseMethod[];
  userType: 'rational' | 'emotional' | 'action' | 'visual';
  personaFrequency: Record<PersonaType, number>;
  personaNicknames: Record<PersonaType, string>;
  preferredScene: SceneType;
  totalThoughts: number;
  totalJournals: number;
  totalChats: number;
  totalSessions: number;
  weeklyLetterDay: 'sunday' | 'saturday';
}

## 九、差异化与竞品分析

### 9.1 与现有产品的差异

| 维度 | 冥想App | 情绪日记App | ChatGPT | 赛博树洞类 | **念头解钩器** |
|------|--------|------------|---------|-----------|-------------|
| 核心机制 | 被动听引导 | 记录情绪 | 对话分析 | 记录+陪聊 | **物化+解钩+陪伴+复盘** |
| 用户角色 | 听众 | 记录者 | 提问者 | 倾诉者 | **观察者+操作者+对话者** |
| 念头处理 | 忽略/接纳 | 分析/归因 | 逻辑反驳 | 倾听/建议 | **物化+解钩+释放** |
| AI能力 | 无/弱 | 弱 | 强但不具象 | 中等 | **分类+改写+人格化+陪伴** |
| 情绪记录 | 无 | 自由/简单 | 无 | 结构化问答 | **12步引导+AI分析** |
| 互动性 | 低 | 低 | 中 | 中 | **高（物理+AI+场景）** |
| 个性化复盘 | 无 | 弱 | 无 | 每周回信 | **每周回信+数据洞察** |
| 即时反馈 | 弱 | 弱 | 中 | 中 | **强（视觉+触觉+AI）** |
| 使用门槛 | 需20min+ | 需文字表达 | 需表达能力 | 中等 | **30秒即可完成一次** |

### 9.2 核心竞争力

1. **具身化解钩**：不只是"想"，而是"看见、触碰、操作"——全网独有
2. **AI四层深度**：分类+改写+人格化+陪伴+复盘——AI贯穿全链路
3. **结构化情绪日记**：12步引导不放过任何一个觉察维度，但不让用户感到压力
4. **场景化陪伴**：5种治愈场景+音效的沉浸式对话体验
5. **个性化复盘**："来自树洞的信"形式的每周回信——温暖而非冷冰冰的数据
6. **即时性**：30秒一个念头，2分钟一次对话，5分钟一篇日记
7. **非诊断性**：不贴标签、不说教，只帮你"看见"和"陪伴"
8. **科学根基**：基于 ACT 认知解离技术 + CBT 情绪日记方法，有循证支持

## 十一、隐私与伦理

### 11.1 隐私原则

- 🔒 **本地优先**：所有数据默认存储在本地设备（念头、日记、对话、信件）
- 🚫 **不上传原文**：AI分析时只发送脱敏/匿名化的内容（或使用本地模型）
- 🗑️ **可删除**：用户随时可以删除所有数据
- 👁️ **透明**：明确告知数据存储位置和AI使用方式
- 🔐 **可选离线**：提供完全离线模式（无AI功能但核心体验可用）

### 11.2 伦理边界

- ⚠️ App 明确声明「不是心理治疗工具」
- 🆘 检测到高风险内容（自残/自杀相关）时，温和引导至专业求助资源
- 💚 不制造焦虑，不设打卡机制，不搞连续使用天数排名
- 🤖 AI结果标注"仅供参考"，不做诊断性结论
- 🧒 念头人格化的角色设计温暖友好，不制造恐惧
- 💬 陪聊AI不鼓励依赖，适时引导用户寻求真人支持
